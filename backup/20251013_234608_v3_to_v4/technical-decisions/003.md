# 决策记录：Turso边缘数据库
**文件**：technical-decisions/003.md
**决策时间**：2025-01-12
**决策原因**：边缘计算 + SQLite性能

## 决策内容
**问题**：选择什么数据库支持边缘计算？
**方案选择**：
- A: PostgreSQL + Prisma
- B: MongoDB + Mongoose
- C: Turso (边缘SQLite)

**选择理由**：
✅ 选择C方案，因为：
1. 边缘SQLite，数据就近存储
2. 全球分布式，低延迟访问
3. SQL语法，学习成本低
4. 符合"无为而治"的分布式理念

**架构优势**：
- 边缘节点：数据就近存储
- 全球同步：自动数据同步
- 低延迟：<50ms响应时间
- 成本效益：按使用量计费

## 实现细节
- 主数据库：Turso Cloud
- 边缘缓存：Turso Edge
- 数据同步：自动同步
- 查询优化：边缘查询优先
